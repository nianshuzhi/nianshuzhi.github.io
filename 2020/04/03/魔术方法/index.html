魔术方法

本文共计4289个字 | 您是第 位看到它们的小伙伴

魔术方法与普通方法相比具有很大的不同。普通方法是根据用户的实现方式进行调用，而魔术方法通常情况下用户不会主动调用，而是在特定的时机被PHP系统自动调用。可以通俗的理解为系统事件监听方法，在事件发生时才触发执行，与嵌入式系统中的中断函数类似。

在PHP中通常以”“打头的方法都作为魔术方法，所以用户不要定位以”“开头的方法。例如类的构造方法，”construct()”，该方法是在创建实例完成后自动调用的一个方法。
对于魔术方法的使用需要了解两个方面的内容，一是魔术方法定义的位置，二是魔术方法调用的时机。
以下是常用魔术方法的使用方法：

construct
构造函数会在每次实例化对象时先调用此方法，所以非常适合在使用对象之前做一些初始化工作
destruct
析构函数会在某个对象的所有引用都被删除或者当对象被显式销毁时执行
set
用户属性重载，在给不可访问的属性赋值时，会被调用
get
读取不可访问属性的值时，会被调用
isset
当对不可访问属性调用isset()或empty时，会被调用
unset
当对不可访问属性调用unset时，会被调用
sleep
serialize()检查类中是否有魔术方法sleep()。如果存在，该函数将在任何序列化之前运行。它可以清除对象并返回一个包含有该对象中应被序列化的所有变量名的数组
wakeup
相反的，unserialize()检查是否具有魔术方法wakeup()。如果存在，此函数可以用于重建对象。
toString
用于一个类被当成字符串时应该怎样回应。例如执行echo $obj;时，此方法会被调用
invoke
当尝试以调用函数的方式调用一个对象时，invoke方法会被自动调用。例如执行$obj;时，此方法会被调用
clone
如果定义了clone魔术方法，则新创建对象时，此方法会被调用，可用于修改属性的值
call
在对象中调用一个不可访问的方法时，会被调用
callStatic
在静态方式调用一个不可访问的方法时，会被调用
__autoload
会在试图使用尚未被定义的类时自动调用

使用场景：比如set和get可以实现在外部调用类的私有变量
